# 商談議事録

## 基本情報

- 顧客名: 株式会社東央ソリューションズ
- 業種: BtoB 向け業務システム導入支援
- 従業員規模: 約160名
- 営業体制: アカウント営業 24名、プリセールス 8名、提案支援 5名
- 日時: 2026-03-12 10:00-11:00
- 参加者:
  - 顧客側: 営業企画部長、提案推進チームリーダー、情報システム担当
  - 提案側: プリセールス、ソリューションアーキテクト

## 案件名

提案業務向けナレッジ検索アシスタント PoC

## 目的

営業担当とプリセールスが、過去提案書や製品資料を横断して素早く参照できるようにしたい。  
案件ごとに必要な情報を探す時間を減らし、提案準備の初速を上げたい。

## 背景

- 提案書、見積前提、製品仕様、過去案件メモが複数の場所に分散している
- 営業が有識者に都度確認しており、提案準備の待ち時間が発生している
- 類似案件の探し方が人によって異なり、再利用の精度にばらつきがある
- 新任メンバーほど、どの資料を見るべきか分からず初動が遅くなりがち

## 現状の課題

- 過去提案や事例を探すのに 20-30 分かかることがある
- 製品資料の版が複数あり、どれを参照すべきか迷いやすい
- 商談メモや提案時の注意点が個人フォルダに閉じている
- 提案のたたき台を作る前段で、情報収集に時間を使いすぎている

## 要望

- 過去提案書、製品説明資料、導入事例メモを横断検索したい
- テキストベースのチャット UI で質問し、関連資料の要点を返してほしい
- 回答時に、どの資料を根拠にしたかを表示してほしい
- 類似案件の提案ポイントや注意事項をすぐ参照できるようにしたい
- PoC では、要約や引用の雰囲気が掴めればよい

## PoC でまず見たいこと

- まずは営業企画部が管理する代表資料 50-100 本程度で試したい
- 参考資料の検索性と、回答の根拠提示が実務で使えそうかを見たい
- UI は PC ブラウザ上のテキストチャットで十分
- PoC の評価指標として、検索時間短縮と回答の参照妥当性を確認したい

## 対象にしたい文書

- 過去提案書
- 製品紹介資料
- 導入事例サマリ
- 提案レビュー時のメモ
- 業界別の提案観点整理資料

## 想定ユーザー

- 営業 20 名
- プリセールス 8 名
- まずは提案初動を担うメンバーから使いたい

## 利用イメージ

- 商談後に「類似案件の提案観点」を検索する
- 提案骨子を作る前に「過去に類似要件でどんな注意点があったか」を確認する
- 製品説明資料から、要件に関係する箇所を抜き出して下書きに活かす

## 制約・懸念

- UI はまずシンプルなチャット形式でよい
- PoC では既存システム連携や SSO は不要
- 回答が断定しすぎず、根拠資料を見ながら使えることを重視したい
- 将来的には提案書ドラフト支援まで広げたいが、今回は検索支援に絞る

## スケジュール感

- 4月前半に PoC を開始したい
- 4月末までに営業企画部で評価したい

## 予算感

- まずは小さめの PoC 予算で進めたい
- 効果が見えれば対象資料やユーザーを広げたい

## 議論メモ

- 顧客としては、完璧な回答生成よりも「まず探しやすいこと」を重視
- 検索時間が短くなるだけでも現場価値が高いとのこと
- 利用ログを細かく見るより、まずは日常業務で自然に使えるかを確認したい
- 提案支援チームとしては、回答の根拠リンクが必須という認識

## 宿題

- 顧客側:
  - PoC 対象にする資料群の候補を 1 セットに絞る
  - 評価指標の候補（検索時間、参照妥当性など）を整理する
  - 利用想定シーンを 3 パターンほど共有する
  - 契約書や提案付帯資料の外部送信可否を情報システム部に確認する
- 提案側:
  - テキストチャット型デモの画面イメージを提示する
  - 資料の投入イメージと検索方式の案を整理する
